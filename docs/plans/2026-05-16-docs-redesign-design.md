# Docs Site Redesign — 2026-05-16

## Goal
Replace the single-page generated docs with a multi-page static site:
1. **Landing page** (`docs/index.html`) — Hero, install steps, feature grid
2. **Rust ↔ zust comparison** (`docs/compare.html`) — Side-by-side code blocks for common patterns
3. **API reference** (`docs/api.html`) — All 58 modules with declarations

## Architecture
- Pure static HTML generated by `docs/generate.zig`
- No JavaScript required
- Single CSS block embedded in each page
- Navigation bar linking all 3 pages

## Landing Page Sections
1. **Hero** — Big title, tagline, GitHub link
2. **Feature grid** — 6 cards: Zero-cost ownership, SIMD acceleration, Static analysis, LSP server, Transpiler, Cross-platform
3. **Install** — One-liner `zig fetch` + build.zig snippet
4. **Quick example** — Hello world with Box
5. **Footer** — Links to compare, API, GitHub

## Comparison Page
Side-by-side tables with:
| Pattern | Rust | zust |
|---------|------|------|
| Heap allocation | `Box::new(42)` | `Box(i32).init(allocator, 42)` |
| Growable array | `Vec::new()` | `ArrayList(T).init(allocator)` |
| Hash map | `HashMap::new()` | `HashMap(K, V).init(allocator)` |
| Shared ownership | `Arc::new(42)` | `Arc(T).init(allocator, 42)` |
| Mutex | `Mutex::new(42)` | `Mutex(T).init(allocator, 42)` |
| String | `String::new()` | `String.init(allocator)` |
| Optional | `Option<i32>` | `?i32` |

Each pattern gets a code block showing Rust on left, zust on right.

## API Reference Page
Same content as current docs but styled better with search/filter box.

## CSS Design
- Dark theme (matches Rust docs aesthetic)
- Orange accent (#e67e22) for zust branding
- Monospace font for code (JetBrains Mono or Fira Code via CDN)
- Responsive layout (max-width 1200px)

## Implementation
- `docs/generate.zig` writes 3 HTML files
- Comparison data is a `comptime` array of structs in `generate.zig`
- API data is parsed from `lib/*.zig` ASTs (existing logic)
- All pages share a `writeHeader()` / `writeFooter()` helper
